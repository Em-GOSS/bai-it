掰it LOGO v3 — 从头想

抛开 v2 的 10 个方向，回到基本原则重新来。 核心认知：掰it 是一个词，不是两个词并排。混合脚本本身就是设计。

掰it 是一个词，不是「掰」+「it」 同色 = 同词，分色 = 分词 光学重量匹配 > 字号匹配 junction 间距 = 字母间距 混合脚本本身就是足够的视觉张力
01
纯字 One Word
掰it
零装饰。同色同重量，紧贴排列。
脚本混搭本身就是设计——中文方块 + 拉丁小写的反差足够抢眼。
最成熟、最自信的表达。
01b
纯字·红 One Word Red
掰it
品牌色版本。全红 = 全是一个词。
用在需要强品牌辨识的场景（启动页、关于页）。
02
徽章 Badge
掰it
Uniqlo 式容器统一。红色圆角矩形把两种脚本锁在一起。
容器做了统一工作，字体本身不需要额外处理。
适合品牌主视觉、商店页面。
02b
徽章·描边 Badge Outline
掰it
描边变体。更轻盈，适合暗色 UI 中融入使用。
框是红的，字是白的——红色做结构不做文字。
03
咬痕 Bite Mark
掰it
掰it ≈ bite。右上角被咬了一口。
把谐音梗做成视觉语言——「咬一口」就懂了。
有记忆点，有故事。
04
掰开 The Split
掰 it
一条红色细缝。不是装饰性裂痕，是精确的排版呼吸。
两部分同色同重，但中间那道缝说着「这里被掰开了」。
克制版的 v2 锯齿裂痕——用 1.5px 线代替 SVG 锯齿。
05
印章 Seal

掰 it
竖排方印。掰在上，it 在下。
天然适配正方形图标（Chrome 扩展、App icon）。
左: 描边版 / 右: 实底版
