# 掰it — 产品需求文档

## 用户痛点

英文阅读中最大的障碍不是单词，而是长句——多层嵌套的从句、倒装、插入语让人一读就晕。现有工具要么只做翻译（失去学习机会），要么只做词典（解决不了句子结构问题）。

更具体地说，用户在两个场景下有不同的痛点：

1. **刷信息流（推特、Reddit）时**：需要快速筛选哪些内容值得细看，但英文长句让扫读速度大打折扣。很多句子语法不复杂，但信息密度高、并列结构多，一眼扫不过去。这不是学习问题，是效率问题。
2. **决定细读一篇文章时**：面对整页英文会被"吓退"。需要适度的辅助把难度降到可接受的范围（i+1 ~ i+3），同时保留足够的挑战让阅读过程产生真正的学习效果。

## 设计原则

1. **装完即用**：用户装插件就能用，不需要 API key、不需要注册。基础功能（拆句 + 生词释义）零配置
2. **数据归用户**：所有数据存浏览器本地，不上传、不泄露
3. **学习而非翻译**：帮用户理解句子结构，不是替用户读
4. **不制造依赖**：工具的目标是让用户逐渐不需要它。扫读提供效率，细读提供成长，两者缺一不可
5. **渐进增强**：基础版本地完成，配了 API key 后体验升级。两层之间平滑过渡，不是两个割裂的产品

## 两层产品模型

掰it 分为基础版和完整版，核心区别在于是否配置了 LLM API key。

### 基础版（无 API key，装完即用）

面向大部分用户。覆盖「看懂」这个最核心的需求：

- **本地拆分**：规则引擎自动拆分长句，零延迟
- **本地生词释义**：离线词典 + 行业术语包，hover 虚线显示中文释义。**这是基础版体验的底线——拆分后的句子中，生词必须有释义**
- **管理端**：只展示一个引导页，告诉用户配置 API key 后能解锁什么（句式分析、难句集、每日回味等），提供配置入口
- **不需要任何配置**，装完就能在英文网页上使用

### 完整版（配了 API key）

面向想深度学习的用户。在基础版之上增加：

- **LLM 深度分析**：句式分类、句式讲解、表达技巧、语境化生词释义
- **LLM 比例提高**：扫读模式下更多句子走 LLM 分析（降低 needsLLM 的触发门槛），积累更多学习数据
- **手动触发**：扫读和细读模式下都显示手动触发按钮，用户可以对任何句子随时触发 LLM 深度分析
- **管理端全功能解锁**：总览、难句集、每日回味、设置四个 Tab 全部可用
- **学习数据自动积累**：LLM 处理过的句子自动存入 IndexedDB，供管理端展示

## 核心概念：两种阅读模式

掰it 区分两种阅读场景，提供不同级别的辅助：

### 扫读模式（Scan Mode）

- **目标**：快速理解，不卡住。服务效率，不服务学习
- **触发**：信息流页面自动启用（推特时间线、Reddit 列表等）
- **分块策略**：本地优先，LLM 按需增强
  - **所有句子：本地规则即时拆分**（零延迟、零 API 成本）
    - 按长度拆：超过阈值的句子都拆（阈值可调：短/中/长三档，默认中）
    - 在逻辑转换点断行：并列、转折、条件、因果、补充、从句引导等
  - **基础版**：本地拆分 + 本地词典释义，不调 LLM
  - **完整版**：降低 LLM 触发门槛，更多句子自动走 LLM 分析；同时显示手动触发按钮，用户可对任何句子点击触发深度分析
- **排版**：保留缩进层级，体现句子内部结构（哪些行属于同一句话）
- **生词**：hover 虚线提示，不直接显示中文释义（避免视觉干扰）
  - **基础版**：离线词典 + 行业术语包标注（V1 内置 AI 行业包）。**本地拆分的结果中，生词释义必须正常显示**
  - **完整版**：LLM 返回的语境化释义覆盖本地释义（更准确）
  - 用户标记"已掌握"的词自动跳过

### 细读模式（Deep Read Mode）

- **目标**：读进去，有成长。把 i+10 的难度降到 i+3，剩下的由用户自己啃
- **触发**：内容页面自动启用（文章页、推特单条详情、Reddit 帖子页等）
- **分块策略**：适度拆分，只拆真正复杂的句子
  - 沿用规则引擎判断复杂度，但提高句子长度的权重（避免信息密度高但语法简单的长句漏掉）
  - 简单长句不自动拆，留给用户自己读
  - **基础版**：未被自动拆的句子不提供手动触发（没有 API key 无法调用 LLM）
  - **完整版**：未被自动拆的句子提供手动触发按钮（hover 可见，点击展开 LLM 分块）
- **进入页面辅助**：生词汇总面板，列出本文难词，降低首次接触的心理压力
- **生词**：hover 虚线提示 + 可标记收录到生词本

### 模式切换

- **自动判断**（默认）：按页面类型（URL 模式 + DOM 结构）自动选择模式
  - 信息流页面（推特首页、Reddit 子版块列表等）→ 扫读模式
  - 内容页面（文章、帖子详情等）→ 细读模式
- **内容长度微调**：信息流中特别长的内容（如推特长 thread），自动提升辅助力度
- **手动兜底**：popup 中提供开关，用户随时可以控制当前页面和站点级别的启用/禁用

### Popup 设计

Popup 是用户日常使用的快捷控制面板，只放高频操作，不放 LLM 配置等低频设置。

**控制区（顶部）：**

- **大按钮**：当前页面的一键开关。两种状态切换——"显示原文"（当前正在拆分，点击可关闭）/ "拆分显示"（当前未拆分，点击可开启）
- **站点级开关**：一行文字"总是拆分 x.com" + toggle 开关。控制整个域名是否启用，关闭后大按钮变灰不可用。默认开启

**拆分设置区：**

- **辅助力度**（一个滑杆，1-5 档）：控制拆多少句子。底层合并了两个参数——扫读模式的拆分粒度（chunkGranularity）和细读模式的自动拆分门槛（sensitivity），用户只看到一个"少拆 ↔ 多拆"的滑杆。每档配一句话说明（如"适中 — 复杂句自动拆，简单句不打扰"）
- **显示方式**（分段选择器 + 预览区）：控制拆完后怎么显示。三个选项：
  - **详细**（对应 L5）：分行 + 缩进，一眼看清主从关系
  - **简洁**（对应 L3）：只分行，不加额外标记
  - **轻微**（对应 L1）：不分行，次要部分变淡
  - 选项名称用干预程度命名（详细/简洁/轻微），下方显示一句话说明 + 实时预览效果

**底部：**
- "更多设置 →" 链接，跳转到 Options 页的设置 Tab

**原型文件**：`mockup-popup.html`（可在浏览器中打开交互体验）

## 句式分类体系

LLM 每次处理复杂句子时，同时返回该句子的句式分类。分类从以下固定列表中选取（LLM 返回英文 key，前端映射为用户友好的名称 + 一句话解释）：

| key | 用户看到的名称 | 一句话解释 |
|-----|--------------|-----------|
| `insertion` | 插入补充 | 句子中间插了一段来补充说明 |
| `background_first` | 先说背景 | 先交代条件/原因/时间，再说正事 |
| `nested` | 层层嵌套 | 句子里套句子，要剥好几层 |
| `long_list` | 一口气列举 | 连着列了好几样东西 |
| `inverted` | 倒着说 | 正常语序被颠倒了 |
| `long_subject` | 超长主语 | 主语说了一大串才到动词 |
| `omission` | 省略跳跃 | 省掉了一些词，要自己补上 |
| `contrast` | 对比转折 | 前后两半在对比或转折 |
| `condition` | 条件假设 | 如果……就……，但说法不直白 |
| `long_modifier` | 超长修饰 | 用一大段话来描述一个东西 |

设计原则：
- 名称用通俗但稳定的措辞，用户能看懂，同时能作为筛选标签归类
- LLM 在 prompt 中被约束只能从以上列表选取，保证标签收敛
- 前端映射表与 prompt 解耦，改措辞不需要改 prompt
- 兜底：万一 LLM 返回列表外的值，显示为"其他"

## 功能范围

### P0：基础体验（零配置可用）

- 两种阅读模式（扫读 + 细读），自动切换
- 本地规则引擎拆分（零延迟、零 API 成本）
- **本地生词释义**：离线词典 + 行业术语包，本地拆分后的结果中生词必须有中文释义显示
- 分块结果渲染（保留缩进层级）
- 全站点支持 + 主流站点专用选择器（X、Reddit、Medium 等）

### P0.5：LLM 增强（需配置 API key）

- 插件直调 LLM API（Gemini 格式 + OpenAI 兼容格式）
- 缓存机制（同一句子不重复调用 API）
- LLM 处理复杂句子时同时返回：分块结果、句式分类、句式讲解、学会表达、生词释义、学习价值标记
- 扫读模式下提高 LLM 触发比例，积累更多学习数据
- 扫读和细读模式下均提供手动触发按钮，用户可随时对任何句子触发 LLM 深度分析

### P1：学习系统

#### 数据采集策略

- **静默采集**：用户阅读过程中自动积累数据，不需要手动操作
- **只记有价值的**：仅记录 LLM 处理过的复杂句子（有分析内容的），本地规则处理的简单句子不存储
- **生词遭遇记录**：每次生词出现自动记录一条遭遇（词、上下文句子、来源 URL、时间），用于频率统计和上下文回顾

#### 难句卡片内容

每条 LLM 处理过的句子存储为一张难句卡片，包含以下层级：

1. **原句**（收起状态可见）
2. **句式标签**（收起状态可见，如"插入补充"，hover 显示一句话解释）
3. **句子分块**（展开后可见，结构拆解）
4. **句式讲解**（用大白话解释这句话为什么难读）
5. **学会表达**（从句中提取一个可套用的表达模板 + 简化仿写示例，目标是让用户学会用类似结构回复/写作）
6. **生词**（词 + 语境释义 + 标记掌握按钮）
7. **来源 URL**

#### 管理界面（插件 Options 页，React）

整体视觉风格：讲究的字体组合（有辨识度的标题字体 + 清晰的正文字体）、克制的配色（3-5 色）、交错入场动画（自定义缓动曲线）、抽象 CSS 装饰（渐变圆、几何线条，不用插图）。

**根据是否配置 API key，管理端展示不同内容：**

**未配置 API key（基础版）：**
- 只展示一个引导页
- 简洁地说明配置 API key 后能解锁什么：句式分析、难句集、每日回味、更精准的生词释义
- 提供 API key 配置入口（内嵌在引导页中，不需要跳到单独的设置页）
- 配置完成后自动切换到完整版界面

**已配置 API key（完整版）：四个 Tab**

**1. Dashboard**
- 统计卡片（遇到的难句数、生词数、已掌握词数、今日新增）
- 最近的难句（每条显示句式标签 + 截断原文）
- 高频生词（按遭遇次数排序）
- 本周回顾（合并在 Dashboard 内，不单独成页）：LLM 生成的周总结 + 常见句式分布 + 词汇精选。按需生成——用户打开 Dashboard 时触发，生成后缓存，同一周内不重复生成

**2. 每日回味**
- 设计理念：**回味，不是复习。** 不制造队列、不打卡、不评分、不记连续天数。用户打开看一眼，10 秒搞定，没有"你还没做完"的压力
- 一屏完事，不翻页

- **断句练习（核心）**
  - 展示一个裸句（无任何标记的原始英文长句）
  - 用户凭语感点击句子中觉得该断开的位置，加分隔标记
  - 点"看答案"，显示 AI 当初的分块结果，用户自行对比
  - **不打分、不判对错。** 用户自己感受"我断对了"还是"原来应该在这里断"
  - 训练的是断句直觉，不是记忆某个句子怎么断

- **选句逻辑**
  - LLM 分块时同时返回一个"学习价值"布尔标记（有嵌套结构、典型句式 → 值得练；纯并列堆砌、太短 → 不值得）
  - 从当天标记为"值得练"的句子中随机挑一个
  - 当天没掰过句子 → 从历史中捞一个没练过的
  - 每次出新句子，不重复旧句子（断句练的是感觉不是记忆，新句子比重复旧句子更有价值）

- **高频词汇（附带）**
  - 从当天所有掰过的内容中独立挑选 3-5 个高频词，不跟句子绑定
  - 可标记"已掌握"，下次自动跳过
  - 按遭遇频次排序，优先展示反复遇到但还没标记掌握的词

- **轻量数字**
  - "你这周掰了 X 句"——成就感，不是压力

- **明确不做的事**：进度条、对错评分、连续天数/打卡、队列（"还剩 N 个未复习"）、间隔重复算法

**3. 难句集**
- 所有 LLM 处理过的复杂句子，按时间倒序排列
- 收起状态：原句（截断）+ 句式标签 + 来源域名 + 时间 + 生词数
- 展开状态：完整的难句卡片内容（分块、讲解、表达、生词）
- 筛选：按句式标签筛选（如只看"层层嵌套"类的句子）
- 搜索：按关键词搜索原句或生词

**4. 设置**
- LLM 提供商和模型选择
- API key 配置（加密存储）
- 分块灵敏度调整
- 其他偏好

#### 新用户 / 未配置 API key 状态

**未配置 API key 时**：管理端只展示引导页（见上方"管理界面"章节），不展示四个 Tab。

**已配置 API key 但无数据时**：各 Tab 展示空状态引导文案，引导用户去浏览英文网页积累数据。

#### 已知词过滤

- 用户在难句卡片或阅读中标记"已掌握"的词，自动从后续页面的生词标注中排除
- 数据存 IndexedDB，Content Script 启动时加载已知词列表

### P2：壁纸

- Options 页内 Canvas 渲染，浏览器端生成 PNG
- 批量生成 + 下载 ZIP
- 手动传输到手机（V1 不做云端传输）

## 不做的事情（V1）

- 云端同步、用户认证、登录系统
- 移动端原生应用
- 多语言翻译目标（V1 仅支持中文释义）
- Safari / Firefox 扩展
- 手动粘贴文本分析（旧版"深度学习"页面，已去掉）
